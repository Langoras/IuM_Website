Syntax

Diese Seite erklärt die grundlegende Syntax und die wichtigsten Befehle in CSS.

Innerhalb dieser Regel legen Deklarationen mit Eigenschaften und Werten fest, wie die ausgewählten Elemente gestaltet werden

Wie ist eine CSS-Regel aufgebaut?

Eine CSS-Regel besteht aus einem Selektor und einem Deklarationsblock.

p {
    color: red;
    font-size: 20px;
}

Der Selektor p wählt Absätze aus. Im Block stehen Eigenschaften wie color und font-size sowie deren Werte.

Wichtige Arten von Selektoren

  • Element-Selektor: z. B. p, h1
  • Klassenselektor: .box
  • ID-Selektor: #header

Diese Selektoren reichen für einfache Webseiten völlig aus.

Eigenschaften und Werte

  • color – Textfarbe
  • background-color – Hintergrundfarbe
  • font-size – Schriftgröße
  • margin – äußerer Abstand
  • padding – innerer Abstand
  • border – Rahmen

Kommentare in CSS

/* Das ist ein Kommentar */

Kommentare helfen, den Code übersichtlich und verständlich zu halten.

Zusammenfassung

CSS beschreibt das Aussehen von Webseiten. Regeln bestehen aus Selektoren und Eigenschaften. Mit CSS steuert man Farben, Abstände, Schriften und Layouts.

Weitere Beispiele für CSS

CSS bietet viele Möglichkeiten, Elemente auf einer Webseite zu gestalten. Neben Farben und Schriften kann man auch den Abstand, die Größen und die Platzierung von Elementen beeinflussen. Ein wichtiger Teil davon ist das Box-Modell. Das Box-Modell beschreibt, wie viel Platz ein Element einnimmt und wie es im Layout dargestellt wird. Jedes HTML-Element besteht aus Inhalt, Padding, Border und Margin. Wenn man versteht, wie diese Bereiche zusammenhängen, kann man Layouts sehr präzise steuern.

Ein weiterer wichtiger Bereich ist die Positionierung. Mit Eigenschaften wie position: relative, position: absolute oder position: fixed kann man Elemente gezielt an bestimmten Stellen der Seite platzieren. Diese Eigenschaften werden in vielen modernen Webseiten verwendet, um Navigationsleisten, Buttons oder Bilder flexibel anzuordnen. Auch die Verwendung von display, insbesondere display: flex und display: grid, spielt eine große Rolle für moderne Layouts. Diese Befehle ermöglichen es, Elemente nebeneinander anzuordnen, auszurichten und flexibel zu skalieren.

Ein typisches Beispiel für Flexbox sieht so aus: 

.container {
    display: flex;
    justify-content: center;
    align-items: center;
}

In diesem Beispiel wird ein Container zu einem flexiblen Bereich, in dem der Inhalt sowohl horizontal als auch vertikal zentriert wird. Flexbox ist besonders praktisch, wenn man Navigationen oder Layouts erstellen möchte, die sich automatisch an verschiedene Bildschirmgrößen anpassen.

Mehr über Farben und Einheiten

Farben in CSS lassen sich auf viele Arten schreiben. Neben einfachen Farbnamen wie red oder blue gibt es Hex-Werte wie #ff0000, RGB-Werte wie rgb(255, 0, 0) und sogar HSL-Werte wie hsl(0, 100%, 50%). Jede dieser Schreibweisen hat ihre Vorteile. Hex-Werte sind sehr kompakt, RGB-Werte eignen sich gut für Bildschirme, und HSL-Werte sind besonders intuitiv, wenn man Helligkeit und Sättigung gezielt einstellen möchte.

Auch bei Einheiten gibt es verschiedene Möglichkeiten. Pixel (px) sind am einfachsten zu verstehen, weil sie eine feste Größe haben. Relative Einheiten wie em oder rem passen sich hingegen an die Schriftgröße an, wodurch Designs flexibler werden. Für responsive Webseiten werden häufig Prozentwerte oder viewport-basierte Einheiten wie vw (Viewport-Breite) und vh (Viewport-Höhe) eingesetzt.

Ein Beispiel für eine responsive Schriftgröße wäre: 

h1 {
    font-size: 5vw;
}

Die Schrift passt sich automatisch der Bildschirmbreite an. Das ist besonders nützlich für Überschriften, die auf kleinen Geräten nicht zu groß und auf großen Bildschirmen nicht zu klein erscheinen sollen.

Für größere Projekte ist es wichtig, CSS gut zu strukturieren. Man sollte klare Klassennamen verwenden und Bereiche logisch trennen. Moderne Methoden wie BEM (Block – Element – Modifier) helfen dabei, den Überblick zu behalten.

Zusätzlich gibt es CSS-Frameworks wie Bootstrap oder Tailwind CSS, die viele fertige Klassen anbieten. Sie können Zeit sparen, aber es ist wichtig, zuerst die Grundlagen gut zu verstehen. Nur so kann man später eigene Designs erstellen und flexibel auf Probleme reagieren.
